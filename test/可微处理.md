# 时序算子可微性处理完整指南（金融因子挖掘专用）

## 一、先明确：哪些算子真的不可微？
**纠正一个常见误解**：**移动平均(ts_mean)、移动求和(ts_sum)、移动标准差(ts_std)、移动相关系数(ts_corr)、移动差分(ts_delta)、移动滞后(ts_lag)都是完全可微的**，可以直接用于梯度优化，不需要任何处理。

真正不可微的时序算子只有以下几类：
- ✅ 可微：ts_mean, ts_sum, ts_std, ts_corr, ts_delta, ts_lag, ts_log, ts_sqrt
- ❌ 不可微：ts_max, ts_min, ts_median, ts_rank, ts_quantile, sign, abs

## 二、通用处理方式：训练用平滑替身，推理用原算子
这是学术界和业界的标准做法，**完全保留因子表达式的可解释性**，只在梯度计算阶段做临时替换。

### 核心原则
- **训练阶段（梯度计算）**：将不可微算子替换为对应的**平滑可微近似版本**
- **推理/回测/入库阶段**：换回**原始硬算子**，因子表达式和计算结果完全不变
- **最终写入库的因子**：仍然使用原始算子名，如`ts_max(close, 20)`，不出现任何"soft"字样

## 三、各不可微算子的具体平滑实现
以下是金融因子挖掘中最常用、数值最稳定的平滑近似方案，全部基于PyTorch实现。

### 1. 移动最大(ts_max)和移动最小(ts_min)
使用**Softmax加权平均**作为平滑近似，这是目前效果最好、最稳定的方法。

#### 公式
```python
def soft_ts_max(x, window_size, k=10.0):
    """
    平滑移动最大值
    x: 输入时序张量，shape=(T, N)，T是时间步，N是品种数
    window_size: 窗口大小
    k: 平滑系数，k越大越接近真实max，k越小越平滑
    """
    # 滑动窗口展开
    x_unfold = x.unfold(0, window_size, 1)  # shape=(T-window_size+1, N, window_size)
    
    # 计算softmax权重
    weights = torch.softmax(k * x_unfold, dim=-1)
    
    # 加权平均得到平滑最大值
    soft_max = torch.sum(weights * x_unfold, dim=-1)
    
    # 填充前window_size-1个值
    padding = torch.full((window_size-1, x.shape[1]), float('nan'), device=x.device)
    return torch.cat([padding, soft_max], dim=0)

def soft_ts_min(x, window_size, k=10.0):
    """平滑移动最小值"""
    return -soft_ts_max(-x, window_size, k)
```

#### 参数选择
- **k值**：建议在5~20之间选择。k=10是一个很好的默认值，平衡了近似精度和梯度稳定性。
- **k值越大**：越接近真实的max/min，但梯度会变得更尖锐，容易出现梯度爆炸。
- **k值越小**：越平滑，梯度越稳定，但近似误差会增大。

### 2. 移动分位数(ts_quantile)和中位数(ts_median)
使用**核密度估计**或**排序-softmax**方法，工程上常用以下简化实现：

```python
def soft_ts_quantile(x, window_size, q=0.5, k=10.0):
    """
    平滑移动分位数
    q: 分位数，0.5为中位数
    """
    x_unfold = x.unfold(0, window_size, 1)  # shape=(T-window_size+1, N, window_size)
    
    # 对每个窗口内的值进行排序
    sorted_x, _ = torch.sort(x_unfold, dim=-1)
    
    # 计算分位数位置的权重
    positions = torch.linspace(0, 1, window_size, device=x.device)
    weights = torch.softmax(-k * torch.abs(positions - q), dim=-1)
    
    # 加权平均得到平滑分位数
    soft_quantile = torch.sum(weights * sorted_x, dim=-1)
    
    # 填充前window_size-1个值
    padding = torch.full((window_size-1, x.shape[1]), float('nan'), device=x.device)
    return torch.cat([padding, soft_quantile], dim=0)

def soft_ts_median(x, window_size, k=10.0):
    """平滑移动中位数"""
    return soft_ts_quantile(x, window_size, q=0.5, k=k)
```

### 3. 移动秩(ts_rank)
使用**高斯核加权排序**：

```python
def soft_ts_rank(x, window_size, k=10.0):
    """
    平滑移动秩，输出范围[0, 1]
    """
    x_unfold = x.unfold(0, window_size, 1)  # shape=(T-window_size+1, N, window_size)
    
    # 计算每个值与窗口内其他值的相似度
    diff = x_unfold.unsqueeze(-1) - x_unfold.unsqueeze(-2)  # shape=(T-window_size+1, N, window_size, window_size)
    similarities = torch.exp(-k * torch.abs(diff))
    
    # 计算秩：有多少个值小于等于当前值
    ranks = torch.sum(similarities, dim=-1) / window_size
    
    # 取最后一个值作为当前时刻的秩
    soft_rank = ranks[:, :, -1]
    
    # 填充前window_size-1个值
    padding = torch.full((window_size-1, x.shape[1]), float('nan'), device=x.device)
    return torch.cat([padding, soft_rank], dim=0)
```

### 4. 符号函数(sign)和绝对值(abs)
```python
def soft_sign(x, k=10.0):
    """平滑符号函数"""
    return torch.tanh(k * x)

def soft_abs(x, k=10.0):
    """平滑绝对值函数"""
    return x * soft_sign(x, k)
```

## 四、窗口大小的可微化处理
时序算子的窗口大小（如5、10、20）是离散整数，本身也是不可微的。标准处理方法是使用**直通估计器(STE, Straight-Through Estimator)**：

```python
class STEWindow(torch.autograd.Function):
    @staticmethod
    def forward(ctx, w):
        """前向传播：取整到最近的整数"""
        return torch.round(w).long()
    
    @staticmethod
    def backward(ctx, grad_output):
        """反向传播：梯度直接传递，不做修改"""
        return grad_output

# 使用方法
window_param = torch.tensor(20.0, requires_grad=True)
window_size = STEWindow.apply(window_param)
factor = ts_mean(close, window_size)
```

**优化结束后**，将`window_param`取整到最近的整数，写回因子表达式即可。

## 五、工程实现关键注意事项
1. **数值稳定性**：
   - 在softmax计算中添加一个小的epsilon：`torch.softmax(k * x_unfold + 1e-8, dim=-1)`
   - 避免k值过大（不超过50），否则会出现数值溢出

2. **梯度消失/爆炸**：
   - 对平滑算子的输出进行标准化：`(x - x.mean()) / (x.std() + 1e-8)`
   - 使用梯度裁剪：`torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)`

3. **算子替换的自动化**：
   - 在你的GP框架中添加一个`use_soft_ops`开关
   - 当`use_soft_ops=True`时，自动将所有不可微算子替换为平滑版本
   - 当`use_soft_ops=False`时，使用原始算子

## 六、处理后的因子表达式需要改变吗？
**完全不需要改变**。

你最终写入库的因子表达式仍然是：
```
w1 * ts_max(close, 20) - w2 * ts_min(volume, 10)
```

而不是：
```
w1 * soft_ts_max(close, 20, k=10) - w2 * soft_ts_min(volume, 10, k=10)
```

**平滑近似只存在于梯度优化的过程中**，不会出现在最终的因子表达式里。这保证了因子的可解释性完全不受影响，同时让你能够利用梯度下降来优化因子的参数和结构。
